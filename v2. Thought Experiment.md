## 📚 HACKER'S DICTIONARY - Simple Version

### Before We Start: Basic Words You Need to Know

---

### 🌐 What is a NETWORK?

**Simple Definition:** Computers connected together so they can share information.

**Easy Analogy:** 
- Your home WiFi is a network
- Your phone, laptop, and smart TV all talk to each other
- At work: All computers in the building connected together

**Why this matters:** Hackers want to get INTO the network to access data.

---

### 🏠 What is an IP ADDRESS?

**Simple Definition:** A unique address for each computer, like a house number.

**Example:**
```
Your house: 123 Main Street
Your computer: 10.50.30.142
```

**You need to know:** Every computer has a unique number so you can find it on the network.

---

### 🚪 What are PORTS?

**Simple Definition:** Virtual doors on a computer where different programs listen.

**Easy Analogy:**
```
IP Address = The building address (123 Main Street)
Port = The door number (Front door, Back door, Side door)

10.50.30.15:80 = "Go to building 15, use the front door (web traffic)"
```

**Common Ports (Just Remember These 3):**

| Port | What It's For | Easy Example |
|------|---------------|--------------|
| **80** | Websites | Front door - public access |
| **22** | Remote login (SSH) | Back door - staff only |
| **3306** | Database | Vault - should be locked! |

**Simple Rule:** 
- Open port = Unlocked door ✅
- Closed port = Locked door 🔒

---

### 🔍 What is NMAP?

**Simple Definition:** A tool that helps you see what computers are on a network and what "doors" (ports) are unlocked.

**Think of it as:** 
- A flashlight that shows you all the houses on a street
- Tells you which doors are open
- Shows you what's inside

**What Nmap Does:**
1. Finds computers on the network
2. Shows which ports (doors) are open
3. Tells you what programs are running

---

### 🎯 Quick Visual: How It All Works Together

```
You plug into the network
        ↓
Use nmap to look around
        ↓
Find computers (IP addresses)
        ↓
Check which doors are open (ports)
        ↓
See what's running (services)
```

### 🔥 **: The Fire Inspector**

**Your Cover Story:**

**Props You Need:**

- ❌ Someone asks for credentials

**Backup Plan:**
- "Oh, let me check my email for the confirmation..."
- "My supervisor sent the notice last week, maybe it got lost?"
- If caught, apologize and leave (you tried!)

## 💻 PART 2: You're Inside - Simple Nmap Basics (USE 127.0.0.1 instead of the test IP Address)

### **Congratulations! You Made It Inside!**

**Scenario:**
- ✅ You successfully talked your way in
- ✅ You're now on the 3rd floor
- ✅ You found an empty conference room
- ✅ There's an ethernet cable in the wall

**Now what?**

---

### Step 1: Connect to the Network

**Physical Connection:**
```
1. Plug your laptop into the ethernet port
2. Wait for connection (little icon shows you're connected)
3. You're now on their internal network!
```

**Check Your Connection:**

```bash
# WHAT THIS DOES: Shows you YOUR computer's address on the network
# Think: "What's my apartment number in this building?"

ip addr show
# Or on Windows: ipconfig
```

**You See:**
```
IP Address: 10.50.30.142
Subnet: 10.50.30.0/24
Gateway: 10.50.30.1

Translation in plain English:
- You're connected!
- You're on the "30" network (like being on the 3rd floor)
- There are 254 possible computers on this network
- The gateway (.1) is like the main door to other floors
```

---

### Step 2: Look Around - Who Else is Here?

**Your First Nmap Command:**

```bash
# WHAT THIS DOES: Finds all computers that are turned on
# Think: "Walk down the hallway and see which office lights are on"

nmap -sn 10.50.30.0/24
```

**Breaking It Down:**
- `nmap` = The tool you're using (your flashlight)
- `-sn` = "Just tell me who's here, don't knock on doors yet"
- `10.50.30.0/24` = "Check every address from .1 to .254"

**What You See:**
```
Nmap scan report for 10.50.30.1
Host is up (0.00041s latency).
MAC Address: AA:BB:CC:DD:EE:01

Nmap scan report for 10.50.30.15
Host is up (0.00092s latency).
MAC Address: AA:BB:CC:DD:EE:15

Nmap scan report for 10.50.30.20
Host is up (0.00076s latency).
MAC Address: AA:BB:CC:DD:EE:20

[... 22 more computers ...]

Nmap done: 256 IP addresses (25 hosts up) scanned in 3.21 seconds
```

**Translation:**
- Found 25 computers that are turned on
- `.1` is probably the router (main door)
- `.15` and `.20` look interesting (maybe servers?)
- This took 3 seconds

---

### Step 3: Check What's Behind the Doors

**Your Second Nmap Command:**

```bash
# WHAT THIS DOES: Knocks on doors and asks "What are you?"
# Think: "Check which doors are unlocked and see what's inside"

nmap -sV 10.50.30.15
```

**Breaking It Down:**
- `-sV` = "Tell me what VERSION of software is running"
- `10.50.30.15` = "Only check this ONE computer"
- This takes longer but gives more detail

**What You See:**
```
Nmap scan report for 10.50.30.15
Host is up (0.00034s latency).

PORT     STATE SERVICE   VERSION
22/tcp   open  ssh       OpenSSH 8.2
80/tcp   open  http      nginx 1.18.0
3306/tcp open  mysql     MySQL 8.0.28

Nmap done: 1 IP address (1 host up) scanned in 8.52 seconds
```

**Translation in Simple Terms:**
```
This computer has 3 doors (ports) unlocked:

Door 22 (SSH): Remote login - like a back door with a key
Door 80 (HTTP): Website - like the front door
Door 3306 (MySQL): DATABASE - like the vault!

The vault is UNLOCKED! 🚨
```

---

### 🎯 What You Discovered

**The Big Finding:**

| What You Found | Why It Matters | What It Means |
|----------------|----------------|---------------|
| **Port 3306 Open** | This is a DATABASE | All the bank's data might be here! |
| **MySQL 8.0.28** | This is the software version | You could look up if this has vulnerabilities |
| **No password required?** | Try connecting: `mysql -h 10.50.30.15` | You might get instant access to ALL data |

**This is BAD security!**
- Databases should NOT be accessible from office computers
- This is like leaving the vault door open
- Anyone on the network can try to connect

## 📊 Simple Nmap Cheat Sheet

**The Only Two Commands You Need to Remember:**

### Command 1: Discovery
```bash
nmap -sn [IP Address]/24
```
**What it does:** Finds all computers on the network  
**When to use:** First step - map the network  
**Time:** Very fast (2-5 seconds)  

### Command 2: Identification
```bash
nmap -sV 10.50.30.15
```
**What it does:** Shows what programs are running  
**When to use:** After discovery - investigate interesting computers  
**Time:** Slower (5-15 seconds per computer)  

**That's it! These two commands are enough for basic reconnaissance.**
